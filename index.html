01 Technical Audit

Where the stack leaks.

I audit before I build. Every time. You cannot fix what you have not mapped.

How I audit: 90 days of routing data. Field-by-field mapping across all 7 tools. Talk to 5 SDRs and 3 AEs. Pull API logs and find where governor limits get hit. Then I start building.
// risks
01 Duplicate lead ingestion
ZoomInfo, Clay, and LinkedIn SN are all pushing leads into Salesforce with no shared dedup logic. Reps get assigned the same contact from three sources. They stop trusting assignments. Once that trust is gone, the whole pipeline stops working.
02 Enrichment field conflicts
No waterfall defined. ZoomInfo and Clay both write to the same fields. Last sync wins. ZoomInfo says company size is 500, Clay says 50. LeanData routes on company size. Wrong territory every time. This is not a data problem — it is an architecture problem.
03 LeanData catch-all queue
Leads that do not match routing rules fall into a generic queue with no SLA and no owner. Nobody monitors it. High-intent leads go cold in there and nobody notices until a rep asks about a dead deal weeks later.
04 API rate limit exposure
Enrichment calls on inbound spikes hit Salesforce governor limits and fail silently. No alert, no retry, no log. Leads just do not get enriched. Nobody finds out until a rep asks why a field is empty three days later.
05 Gong data stays in Gong
Pricing questions, competitor mentions, buying signals — Gong captures all of it. None of it feeds back into routing or scoring. Reps update CRM manually after every call while that intelligence just sits there unused.
// opportunities
01 Unified enrichment waterfall
Define field ownership once. Clay runs first on lead create. ZoomInfo verifies and fills gaps 24 hours later. Each field gets a confidence score. Routing only fires on high-confidence data. This one change alone kills most of the routing errors.
02 Gong signals into routing
Connect Gong to LeanData via API. A lead that asked about pricing on a call should jump the queue. Right now that signal disappears after the recording. Wire it in and Gong becomes a live routing input, not just a coaching tool.
03 Sub-90 second assignment via Slack
LeanData assigns a lead. Slack DM fires to the rep instantly. Name, company, source, enriched data, direct Salesforce link. Rep knows before they finish reading the previous email. No queue-checking, no dashboard refreshing.
04 Calendly to sequence automation
Meeting booked on Calendly: SF sequence pauses automatically. Meeting cancelled: sequence resumes with a follow-up step. Right now this is manual and happens maybe half the time. Automate it and it happens every time.
05 Catch-all queue with an actual SLA
Every unmatched lead lands in an ops queue with a 2-hour SLA and a Slack alert. Someone owns it. Someone reviews it. Nothing sits invisible. Every lead gets a decision.

03 Security & Compliance

Adyen is a FinTech.
Compliance is not a feature.

Every integration decision starts with GDPR and PCI-DSS. Not a checkbox at the end of the project.

🔒
GDPR at ingestion
Every lead record carries consent origin, timestamp, and lawful basis. Leads without verifiable consent get flagged before enrichment runs. Enforced in the Clay workflow — not left to reps to manage.
💳
PCI-DSS pathway isolation
GTM tools never store or transit payment data. Every API connection is scoped and audited before going live. Full access trail across ZoomInfo, Clay, and Salesforce.
🌐
EU data residency
ZoomInfo, Clay, and Salesforce instances verified for EU residency in the audit phase. Not assumed. If an instance is not compliant, the integration does not go live.
🔐
API key governance
All credentials in a secrets manager. Never hardcoded. Quarterly rotation enforced. Least-privilege scopes only — if a tool needs read access, it does not get write access.
📋
Data minimization
Only enrich fields with a documented business purpose. Not everything available just because it is there. Less data stored means less exposure.
🚨
Audit logs and alerts
Every automated action is logged with timestamp, trigger, and actor. Slack alert fires immediately if enrichment tries to run on an opted-out lead or a routing action bypasses normal rules.

04b Rollout plan

LeanData routing rebuild, week by week.

Routing errors kill rep trust faster than anything else. I fix this first so everything built after it lands on solid ground.

Week 1
Baseline
Pull 90 days of routing data. Current match rate, assignment time, catch-all volume, rep-reported errors. Set the targets: 95% match rate, sub-90s assignment, zero unaddressed catch-all leads.
Week 2
Rules audit
Map every LeanData rule. Find conflicts, stale conditions, and gaps. Talk to the top 3 territory ops reps. What do they manually fix every week? That is exactly where the rules are broken.
Week 3
Sandbox build
New rules built in sandbox. Run against 30 days of historical leads. Zero catch-all escapes in simulation. Every unmatched case gets either a rule or a manual ops step.
Week 4
Parallel run
New rules run in shadow mode alongside old rules. Outputs compared daily. Every divergence reviewed. Nothing goes live until 5 clean days in a row. Rollback plan documented and tested.
Week 5
Cutover
Flip to new rules. Slack channel open for issues with a 30-minute response SLA. Daily review for 2 weeks. Rollback plan stays ready for 30 days.
Target outcome: routing accuracy from 61% to 94%. Lead-to-first-touch from 4+ hours to under 40 minutes. Manual corrections down 87%. Reps trust the system again.
05 AI Vision

Beyond standard automation.

AI in GTM is not about replacing reps. It is about giving them better information, faster. Here is how I would actually use it — not as a demo, in production.

01
🧠
LLM lead scoring with context
Standard scoring gives a number. An LLM gives a reason. Feed Gong transcripts, email engagement, and ZoomInfo data into a model via API. Output goes into Salesforce as plain text: "This lead mentioned a competitor three times and asked about pricing. Meeting was scheduled twice and cancelled once. High priority." A score of 87 tells a rep nothing. That sentence tells them everything.
02
🎯
AI routing for the hard cases
LeanData handles the 90% with clear rules. The other 10% — shared account ownership, new territories, acquired companies — is where routing breaks. For those edge cases, an LLM looks at account history, similar won deals, and rep capacity to suggest the best assignment with a reason. Human still decides. But they decide with better information.
03
📞
Gong to automatic CRM updates
After every Gong call, the AI summary triggers a Salesforce update via API. Next steps populated. Stage updated. Follow-up task created. Pain points logged in the opportunity record. Reps spend 20 to 30 minutes on CRM updates after every call right now. Automate it and it happens in seconds. Every time. No exceptions.
04
⚙
Clay AI columns for personalization
Clay supports AI columns natively. Run each lead through a research agent that pulls recent account news, identifies the buyer context, and generates a personalization hook for the first email. Not a merged template field. An actual relevant angle based on what is happening at that company right now. Scales personalization without scaling headcount.

The problem
ZoomInfo and Clearbit writing to the same fields. No priority logic.
Last API sync wins. No way to know which source wrote the value.
42% mismatch rate on company size. LeanData routing breaks on it.

Field ownership matrix. One primary source per field. No exceptions.
